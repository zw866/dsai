# Skill: References section (verified URLs only)

## Where URLs come from

Only these places may supply URLs for **`## References`**:

- The **Server web preflight** section in the user message (starts with `=== Server web preflight`).
- Any **`web_search`** tool message in the thread.
- Text the **user explicitly pasted** in their task (optional one line, labeled).

Inside those blocks, find **`### Retrieved URLs for References`**. Below it you will see numbered lines with **`URL (verbatim):`** and a URL in backticks.

## What to write under `## References`

1. Start the heading exactly: `## References`
2. Numbered list, one entry per retrieved URL:
   - Example: `1. [Con Edison outage update](https://example.com/...)` using the **exact** URL from the block.
3. If the block says **no URLs** were extracted or search is **disabled**:
   - `1. No URLs retrieved from live search in this session (search disabled or empty results).`
4. **Do not** add “URL not provided” placeholders for pages you did not retrieve—omit them.

## Key points vs References

- **Key points** = short synthesis; every factual claim should be traceable to something in preflight, tool output, or user text.
- **References** = the **clickable audit trail**; if you have no URLs, the audit trail is empty and you say so clearly.
